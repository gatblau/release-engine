# Release Engine (Part 11: Connector Design)

## Overview

This document proposes an approach to connector architecture that differentiates connector types and implementations while maintaining minimal complexity.

## Requirements

1. Differentiate between types of connectors (git, cloud, argocd, crossplane, backstage, other)
2. Support multiple implementations per type (cloud → AWS, Azure, Google Cloud)
3. Modules specify connector type-technology combination (e.g., cloud-aws, git-github)
4. No preferred technologies — module defines which technology to use
5. Allow technology replacement without module rebuilds
6. Keep design simple and minimal

---

## Design

### 1. Core Types

```go
// ConnectorType represents the category of connector.
type ConnectorType string

const (
    ConnectorTypeGit   ConnectorType = "git"     // github, gitlab, bitbucket
    ConnectorTypeCloud ConnectorType = "cloud"   // aws, azure, gcp
    ConnectorTypeCD    ConnectorType = "cd"      // argocd, fluxcd
    ConnectorTypeInfra ConnectorType = "infra"   // crossplane, terraform
    ConnectorTypeDevOps ConnectorType = "devops" // backstage
    ConnectorTypeOther ConnectorType = "other"
)

// ValidConnectorTypes is the set of known types for registration validation.
var ValidConnectorTypes = map[ConnectorType]bool{
    ConnectorTypeGit:    true,
    ConnectorTypeCloud:  true,
    ConnectorTypeCD:     true,
    ConnectorTypeInfra:  true,
    ConnectorTypeDevOps: true,
    ConnectorTypeOther:  true,
}
```

**Type-to-tool mapping reference:**

| ConnectorType | Tools |
|---------------|-------|
| `git` | GitHub, GitLab, Bitbucket |
| `cloud` | AWS, Azure, Google Cloud |
| `cd` | ArgoCD, FluxCD |
| `infra` | Crossplane, Terraform |
| `devops` | Backstage |
| `other` | Anything not covered above |

### 2. ConnectorResult and ConnectorError

```go
// ConnectorResult is the standard return type for all connector operations.
type ConnectorResult struct {
    // Status indicates the outcome: "success", "retryable_error", "terminal_error"
    Status string

    // Output holds operation-specific return data.
    Output map[string]interface{}

    // Error holds structured error information when Status is not "success".
    Error *ConnectorError
}

const (
    StatusSuccess        = "success"
    StatusRetryableError = "retryable_error"
    StatusTerminalError  = "terminal_error"
)

// ConnectorError provides structured error information from a connector call.
type ConnectorError struct {
    // Code is a machine-readable error identifier (e.g., "rate_limited", "not_found").
    Code string

    // Message is a human-readable description.
    Message string

    // Retryable indicates whether the caller should retry the operation.
    Retryable bool
}

func (e *ConnectorError) Error() string {
    return fmt.Sprintf("%s: %s", e.Code, e.Message)
}
```

### 3. Connector Interface

All connector implementations must be safe for concurrent use by multiple goroutines.

```go
// Connector is the interface every connector implementation must satisfy.
// Implementations MUST be goroutine-safe.
type Connector interface {
    // Type returns the connector category.
    Type() ConnectorType

    // Technology returns the specific implementation (e.g., "github", "aws").
    Technology() string

    // Key returns the registry key. Format: "type-technology" (e.g., "git-github").
    // This is derived from Type() and Technology() and must equal
    // string(Type()) + "-" + Technology().
    Key() string

    // Validate checks that the given operation and input are well-formed
    // before execution. Returns nil if valid.
    Validate(operation string, input map[string]interface{}) error

    // Execute runs a specific operation with the given input.
    // The context MUST carry a call_id value (see CallIDFromContext).
    // Implementations use the call_id for provider-side idempotency.
    Execute(ctx context.Context, operation string, input map[string]interface{}) (*ConnectorResult, error)

    // Close releases any resources held by the connector (clients, connections).
    Close() error
}
```

### 4. Call ID Context Propagation

The runner places a unique `call_id` into the context before invoking a connector. Connectors use this for provider-side idempotency as described in the external effect lifecycle (d02/d05).

```go
type ctxKey string

const callIDKey ctxKey = "connector_call_id"

// WithCallID returns a new context carrying the given call ID.
func WithCallID(ctx context.Context, callID string) context.Context {
    return context.WithValue(ctx, callIDKey, callID)
}

// CallIDFromContext extracts the call ID from the context.
// Returns ("", false) if not present.
func CallIDFromContext(ctx context.Context) (string, bool) {
    v, ok := ctx.Value(callIDKey).(string)
    return v, ok
}
```

### 5. Base Connector

A base type that handles boilerplate so implementations only need to focus on their logic:

```go
// BaseConnector provides a reusable foundation for connector implementations.
type BaseConnector struct {
    ConnectorType ConnectorType
    Tech          string
}

func (b *BaseConnector) Type() ConnectorType { return b.ConnectorType }
func (b *BaseConnector) Technology() string   { return b.Tech }
func (b *BaseConnector) Key() string          { return string(b.ConnectorType) + "-" + b.Tech }
```

### 6. Registry Implementation

```go
// ConnectorRegistry manages connector registration and lookup.
type ConnectorRegistry interface {
    // Register adds a connector. Returns an error if the key is already registered.
    Register(connector Connector) error

    // Replace registers a connector, overwriting any existing registration for the
    // same key. Intended for testing and hot-reload scenarios.
    Replace(connector Connector) error

    // Lookup returns a connector by its key (type-technology).
    Lookup(key string) (Connector, bool)

    // ListByType returns all connectors of the given type.
    ListByType(connectorType ConnectorType) []Connector

    // Close calls Close() on every registered connector and clears the registry.
    Close() error
}

type connectorRegistry struct {
    mu         sync.RWMutex
    connectors map[string]Connector
}

func NewConnectorRegistry() ConnectorRegistry {
    return &connectorRegistry{
        connectors: make(map[string]Connector),
    }
}

// validateConnector checks structural invariants before any registration.
func validateConnector(c Connector) error {
    if !ValidConnectorTypes[c.Type()] {
        return fmt.Errorf("unknown connector type: %s", c.Type())
    }
    if c.Technology() == "" {
        return fmt.Errorf("connector technology must not be empty")
    }
    expected := string(c.Type()) + "-" + c.Technology()
    if c.Key() != expected {
        return fmt.Errorf("connector key %q does not match expected %q", c.Key(), expected)
    }
    if strings.Contains(c.Technology(), "-") {
        return fmt.Errorf("technology must not contain '-': %s", c.Technology())
    }
    return nil
}

func (r *connectorRegistry) Register(connector Connector) error {
    if err := validateConnector(connector); err != nil {
        return err
    }

    r.mu.Lock()
    defer r.mu.Unlock()

    key := connector.Key()
    if _, exists := r.connectors[key]; exists {
        return fmt.Errorf("connector already registered: %s", key)
    }

    r.connectors[key] = connector
    return nil
}

func (r *connectorRegistry) Replace(connector Connector) error {
    if err := validateConnector(connector); err != nil {
        return err
    }

    r.mu.Lock()
    defer r.mu.Unlock()

    r.connectors[connector.Key()] = connector
    return nil
}

func (r *connectorRegistry) Lookup(key string) (Connector, bool) {
    r.mu.RLock()
    defer r.mu.RUnlock()

    connector, exists := r.connectors[key]
    return connector, exists
}

func (r *connectorRegistry) ListByType(connectorType ConnectorType) []Connector {
    r.mu.RLock()
    defer r.mu.RUnlock()

    var result []Connector
    for _, connector := range r.connectors {
        if connector.Type() == connectorType {
            result = append(result, connector)
        }
    }
    return result
}

func (r *connectorRegistry) Close() error {
    r.mu.Lock()
    defer r.mu.Unlock()

    var errs []error
    for key, connector := range r.connectors {
        if err := connector.Close(); err != nil {
            errs = append(errs, fmt.Errorf("closing %s: %w", key, err))
        }
    }
    r.connectors = make(map[string]Connector)

    if len(errs) > 0 {
        return fmt.Errorf("errors closing connectors: %v", errs)
    }
    return nil
}
```

### 7. Technology Selection

Technology selection is explicit and direct:

1. Modules specify the desired connector in their step configuration
2. The format is `type-technology` (e.g., `git-github`, `cloud-aws`)
3. The registry looks up the connector by this exact key
4. No complex resolution or fallback mechanisms

Example job configuration:

```yaml
steps:
  - name: create_repository
    connector: git-github
    operation: create_repository
    input:
      org: "my-org"
      name: "new-service"

  - name: provision_cluster
    connector: cloud-aws
    operation: create_cluster
    input:
      region: "us-east-1"
      size: "medium"
```

### 8. Connector Implementation Example

```go
// GitHubConnector implements the GitHub-specific git connector.
type GitHubConnector struct {
    BaseConnector
    client *github.Client
}

// NewGitHubConnector constructs a GitHub connector.
// Credentials are injected at construction time via Volta's UseSecret pattern.
func NewGitHubConnector(client *github.Client) *GitHubConnector {
    return &GitHubConnector{
        BaseConnector: BaseConnector{
            ConnectorType: ConnectorTypeGit,
            Tech:          "github",
        },
        client: client,
    }
}

func (g *GitHubConnector) Validate(operation string, input map[string]interface{}) error {
    switch operation {
    case "create_repository":
        if _, ok := input["org"].(string); !ok {
            return fmt.Errorf("input 'org' is required and must be a string")
        }
        if _, ok := input["name"].(string); !ok {
            return fmt.Errorf("input 'name' is required and must be a string")
        }
        return nil
    case "delete_repository":
        if _, ok := input["org"].(string); !ok {
            return fmt.Errorf("input 'org' is required and must be a string")
        }
        if _, ok := input["name"].(string); !ok {
            return fmt.Errorf("input 'name' is required and must be a string")
        }
        return nil
    default:
        return fmt.Errorf("unsupported operation: %s", operation)
    }
}

func (g *GitHubConnector) Execute(ctx context.Context, operation string, input map[string]interface{}) (*ConnectorResult, error) {
    if err := g.Validate(operation, input); err != nil {
        return &ConnectorResult{
            Status: StatusTerminalError,
            Error:  &ConnectorError{Code: "validation_failed", Message: err.Error()},
        }, nil
    }

    callID, ok := CallIDFromContext(ctx)
    if !ok {
        return nil, fmt.Errorf("call_id missing from context")
    }

    switch operation {
    case "create_repository":
        return g.createRepository(ctx, callID, input)
    case "delete_repository":
        return g.deleteRepository(ctx, callID, input)
    default:
        return &ConnectorResult{
            Status: StatusTerminalError,
            Error:  &ConnectorError{Code: "unknown_operation", Message: operation},
        }, nil
    }
}

func (g *GitHubConnector) Close() error {
    // github.Client does not require explicit cleanup
    return nil
}

func (g *GitHubConnector) createRepository(ctx context.Context, callID string, input map[string]interface{}) (*ConnectorResult, error) {
    org := input["org"].(string)
    name := input["name"].(string)

    repo, _, err := g.client.Repositories.Create(ctx, org, &github.Repository{
        Name: github.String(name),
    })
    if err != nil {
        if rateLimited(err) {
            return &ConnectorResult{
                Status: StatusRetryableError,
                Error:  &ConnectorError{Code: "rate_limited", Message: err.Error(), Retryable: true},
            }, nil
        }
        return &ConnectorResult{
            Status: StatusTerminalError,
            Error:  &ConnectorError{Code: "api_error", Message: err.Error()},
        }, nil
    }

    return &ConnectorResult{
        Status: StatusSuccess,
        Output: map[string]interface{}{
            "repo_url":  repo.GetHTMLURL(),
            "repo_id":   repo.GetID(),
            "clone_url": repo.GetCloneURL(),
        },
    }, nil
}
```

### 9. Application Bootstrap

Connectors are wired explicitly at application startup. No `init()` registration.

```go
func main() {
    // ... config, database, volta setup ...

    // Build the registry
    registry := NewConnectorRegistry()

    // Construct connectors with credentials from Volta
    ghToken, err := volta.UseSecret(ctx, "github-token")
    if err != nil {
        log.Fatalf("failed to fetch github token: %v", err)
    }
    ghClient := github.NewClient(oauth2.NewClient(ctx, oauth2.StaticTokenSource(
        &oauth2.Token{AccessToken: ghToken},
    )))

    // Register connectors explicitly
    if err := registry.Register(NewGitHubConnector(ghClient)); err != nil {
        log.Fatalf("failed to register github connector: %v", err)
    }

    awsSession, err := session.NewSession(/* ... */)
    if err != nil {
        log.Fatalf("failed to create aws session: %v", err)
    }
    if err := registry.Register(NewAWSConnector(awsSession)); err != nil {
        log.Fatalf("failed to register aws connector: %v", err)
    }

    // Pass registry to the runner
    runner := NewRunner(registry, /* ... */)

    // ... start HTTP server, scheduler, etc. ...

    // Graceful shutdown
    defer registry.Close()
}
```

### 10. Runner Integration

The runner calls connectors during step execution:

```go
func (r *Runner) executeStep(ctx context.Context, step Step, jobCtx map[string]interface{}) (*ConnectorResult, error) {
    // Look up the connector
    connector, found := r.registry.Lookup(step.Connector)
    if !found {
        return nil, fmt.Errorf("connector not found: %s", step.Connector)
    }

    // Validate before execution
    if err := connector.Validate(step.Operation, step.Input); err != nil {
        return &ConnectorResult{
            Status: StatusTerminalError,
            Error:  &ConnectorError{Code: "validation_failed", Message: err.Error()},
        }, nil
    }

    // Generate call_id for idempotency and inject into context
    callID := fmt.Sprintf("%s-%s-%s", step.JobID, step.Name, uuid.New().String())
    ctx = WithCallID(ctx, callID)

    // Log the call for audit
    if err := r.logConnectorCall(ctx, callID, step); err != nil {
        return nil, fmt.Errorf("failed to log connector call: %w", err)
    }

    // Execute
    result, err := connector.Execute(ctx, step.Operation, step.Input)

    // Update call log with result
    _ = r.updateConnectorCallLog(ctx, callID, result, err)

    return result, err
}
```

## Design Decisions and Trade-offs

### `map[string]interface{}` as the input/output type

This design uses `map[string]interface{}` for connector inputs and outputs. This is an explicit trade-off:

- **Why:** Connectors are a plugin boundary. Different connector types have fundamentally different input shapes. Typed structs per operation per connector would create a combinatorial explosion of types and tight coupling between the registry layer and every implementation.
- **Mitigation:** Every connector must implement `Validate()` to check inputs before execution. The runner calls `Validate()` before `Execute()`. Malformed inputs are caught at validation time, not deep in provider API calls.
- **Future option:** If a connector's operation set stabilises, typed wrapper functions can be added on top of the generic interface without changing the registry or runner.

### No `init()` registration

Connectors are wired in `main()` with explicit construction and credential injection. This ensures:
- Deterministic startup order
- Testability (swap implementations without import tricks)
- Credential injection via Volta happens at a known point in the boot sequence

### No fallback or resolution chains

A module asks for `cloud-aws` and either gets it or fails. There is no mechanism to fall back to a different technology. This is intentional — silent substitution of a cloud provider would be a correctness and security hazard.

### Technology string must not contain hyphens

The key format is `type-technology` split on the first hyphen. Restricting technology names to hyphen-free strings (e.g., `github`, `aws`, `argocd`) avoids ambiguous parsing.

## Alignment with Release Engine Architecture

| Concern | How it maps |
|---------|-------------|
| Runner → Connector → Provider (arrows ⑬/⑭) | `Runner.executeStep` → `Connector.Execute` → provider API |
| Credential management (arrow ㉛) | Volta `UseSecret` at construction time in `main()` |
| Provider-side idempotency (`call_id`) | Propagated via context; connectors extract with `CallIDFromContext` |
| Connector call logging (`connector_call_log` table) | Runner logs before and after `Execute` |
| External effect lifecycle (d02/d05) | `ConnectorResult.Status` maps to effect states; retryable vs terminal |
| GitOps delivery chain | Git connector writes to repo → ArgoCD detects → EKS → Crossplane → Cloud |
